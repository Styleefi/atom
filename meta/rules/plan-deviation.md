# Plan deviations require options, not unilateral action

When work hits a decision that is not in the approved plan — a new tool to install, a scope change, a workaround for an unexpected obstacle — **stop before executing**. Present: (1) what happened, (2) the available options with trade-offs, (3) a recommendation. Let the owner decide.

Executing a deviation unilaterally is forbidden even when the deviation seems obviously right. (Origin: an attempt to install `uv` without explanation when the planned venv path failed, 2026-07-19.)
